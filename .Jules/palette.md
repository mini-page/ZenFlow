## 2026-03-02 - Reusable Accessible IconButton
**Learning:** Forcing `aria-label` as a required prop in the component interface guarantees accessibility compliance and is a powerful way to enforce standards when working with icon-only buttons. Using `React.forwardRef` ensures broader component compatibility.
**Action:** Always wrap generic UI components like `IconButton` in `React.forwardRef` and use strictly typed props for accessibility attributes.
## 2026-04-02 - Dashboard Accessibility Improvements\n**Learning:** When using icon-only buttons or interactive elements like custom inputs embedded within a UI (e.g. the Dashboard 'Save Name' and 'Focus' toggle), it's critical to provide an `aria-label` and keyboard focus styles (using Tailwind's `focus-visible:ring`). Without them, screen readers announce nothing and keyboard users cannot see where their focus is.\n**Action:** Always add `aria-label` and `focus-visible:ring-2` (or similar focus indicators) to interactive elements that lack text content or rely entirely on icons to communicate function.
